# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for MiniBook.

ADRs document significant architectural decisions made during the development of the project.

## Index

| ADR | Title | Status |
|-----|-------|--------|
| [ADR-001](001-plugin-architecture.md) | Plugin Architecture for Output Formats | Accepted |
| [ADR-002](002-security-csp-autoescape.md) | Security: CSP and Jinja2 Autoescape | Accepted |
| [ADR-003](003-url-validation.md) | URL Validation Strategy | Accepted |

## ADR Format

Each ADR follows this structure:

1. **Title**: Short description of the decision
2. **Status**: Proposed, Accepted, Deprecated, or Superseded
3. **Context**: The situation and problem being addressed
4. **Decision**: The decision that was made
5. **Consequences**: The resulting context after applying the decision
